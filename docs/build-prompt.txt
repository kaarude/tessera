NON-NEGOTIABLE PRODUCT RULES
- Notes are private by default.
- Notes may only be shared when the owner explicitly enables sharing.
- Sharing options:
  - whole team
  - specific users
- Notes must be saved in cloud storage and also downloadable as exact `.md`
  files.
- Notes must remain real markdown, not fake rich text.
- Calendar entries and tasks are separate features.
- Do not build email reminders.
- Use browser notifications only.
- Add a small popup/banner on login for important or overdue items.
- Taskboard must support drag and drop.
- Tasks must be reassignable to other users and other teams.
- Right-clicking a task must open an “Assign to” menu with all accessible
  users and all accessible teams.
- Tasks belong to teams and groups.
- Users can belong to multiple teams.
- Roles are fully custom and unlimited.
- Roles are separate per team.
- Every role must be editable through a checkbox permission editor.
- Admins have all permissions.
- Only admin users can grant admin permissions.
- Admins can create users, teams, groups, and roles.
- Audit logs are admin-only.
- Audit logs must be filterable by team, group, user, action, and include an
  “All Audit” view.
- UI must be dark with orange accents.
- Responsive for desktop, tablet, and mobile.
- Users can change their password later.
- Admin-created accounts use email/password.
- Do not ask me unnecessary questions. Make reasonable product decisions and
  continue building.
